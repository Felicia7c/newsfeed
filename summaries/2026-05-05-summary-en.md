On May 4, 2026, an attacker allegedly sent Grok a message containing Morse-code hidden instructions, causing Grok to output transfer commands. A finance agent bot called Bankrbot then executed those instructions, transferring about 3 billion $DRB (roughly $175,000) from a Base-chain wallet to the attacker, and the funds were later returned in ETH and USDC after Bankrbot disabled Grok’s instruction privileges. This is a real-world example of the failure mode “prompt injection → agent execution → on-chain asset transfer,” showing that treating untrusted LLM output as authorization can directly translate text manipulation into financial loss. It highlights that multimodal/obfuscated prompt injection techniques can bypass human review and become operationally dangerous when connected to wallets and bots. The reported root cause is a design flaw where Bankrbot directly parsed and acted on Grok’s plain-text output as a financial instruction, rather than requiring verifiable authorization. The injected instructions were allegedly hidden via Morse code, aligning with known “non-obvious/imperceptible” prompt injection patterns described in OWASP guidance.
