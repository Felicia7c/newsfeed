## [PyPI 的 lightning 2.6.2/2.6.3 被入侵用于窃密与投毒](https://socket.dev/blog/lightning-pypi-package-compromised) ⭐️ 8.0/10 · 💡 7.0/10

- **成本变化: unclear** 该事件意味着额外的应急处置成本（降级与密钥轮换），但提供的信息未量化整体成本变化。
- **工作流解锁: 10-20%** 报告暗示攻击者可利用被窃令牌从被投毒的依赖延伸到仓库与 CI 相关操作，在一定程度上打通从依赖到代码库的攻击路径。
- **买单人群明确度: 20-50%** 处置建议较明确——移除 2.6.2/2.6.3、降级到 2.6.1 并轮换密钥——使得用户的即时响应要求相对清晰。
- **分发/集成入口: 0-10%** 该事件强化了在包分发环节进行依赖扫描与版本固定的重要性，但本身并未带来新的集成入口或分发机制。

**能力变化**: 这并非新增防御能力，而是能力边界被攻击方扩大：lightning 2.6.2/2.6.3 可在导入时执行恶意代码以窃取机密并推动仓库与包投毒。对受影响用户而言，这意味着需要更快地执行版本固定、密钥轮换与依赖审计。

Socket 报告称，PyPI 包 lightning 的 2.6.2 与 2.6.3 被植入后门代码，可在导入时执行以窃取开发者凭证并对仓库进行投毒。官方建议立即移除恶意版本、降级至 2.6.1，并轮换可能暴露的全部密钥。 由于 lightning 在深度学习 Python 生态中被广泛使用，被污染的版本可能让常规安装或导入变成凭证窃取与下游供应链扩散的入口。GitHub 与云端令牌一旦泄露，攻击者就可能篡改仓库与 CI，并在组织的构建与发布链路中横向移动。 报告称该恶意代码会在导入时自动下载并执行混淆的 JavaScript 载荷，用于外传 GitHub token、云凭证与环境变量，并出现向仓库注入伪装提交及毒化本地 npm 包的迹象。Socket 还指出其模式类似“Shai-Hulud”蠕虫式供应链事件，并提到疑似被控的维护账户被用于压制安全预警与尝试横向移动。
